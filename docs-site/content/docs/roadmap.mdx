# Roadmap

This page tracks snpm's feature development. We're building a fast, secure, and user-friendly package manager that matches and exceeds pnpm's feature set.

## Implemented

Features that are already available in the current version of snpm.

### Core CLI
- [x] **Full command suite** (`install`, `add`, `remove`, `run`, `init`, `upgrade`, `outdated`)
- [x] **Production installs** (`--production` flag)
- [x] **Global package support** (`snpm add -g`, `snpm list -g`)
- [x] **Script execution** (`run`, `exec`, `dlx`)
- [x] **Package patching** (`snpm patch edit/commit/remove/list`)
- [x] **Cache management** (`snpm clean`)
- [x] **Configuration display** (`snpm config`)
- [x] **Authentication** (`snpm login`, `snpm logout`)

### Package Resolution
- [x] **Global Store** with parallel downloads
- [x] **Lockfile** (`snpm-lock.yaml`) read/write and reuse
- [x] **Semver support** (including `latest`, `||` ranges, complex expressions)
- [x] **Virtual store layout** (`.snpm/` directory)
- [x] **Multiple link backends** (auto, hardlink, symlink, copy)
- [x] **Configurable hoisting** (none, single-version, all)

### Workspaces
- [x] **First-class workspace support** (`snpm-workspace.yaml`, `pnpm-workspace.yaml`)
- [x] **Local package resolution** (workspace protocol `workspace:*`)
- [x] **Single lockfile** for the entire workspace
- [x] **Targeted commands** (`snpm add -w <project>`, `snpm install -w <project>`)
- [x] **Recursive scripts** (`snpm run build --recursive`, `--filter`)
- [x] **Catalog protocol** (`snpm-catalog.yaml`)

### CI/CD & Security
- [x] **Frozen lockfile mode** (`--frozen-lockfile` / `--immutable`)
- [x] **Minimum version age** protection against zero-day attacks
- [x] **Install script blocking** with explicit whitelist
- [x] **Registry authentication** (Bearer/Basic, scoped registries)

### Protocols
- [x] **npm:** registry packages
- [x] **file:** local filesystem dependencies
- [x] **git:** git repository dependencies
- [x] **jsr:** JSR registry support
- [x] **catalog:** version catalog references

### Quality of Life
- [x] **Lazy install** on `run`/`exec` if deps are stale
- [x] **Verbose logging** (`--verbose`, `SNPM_VERBOSE`)
- [x] **Overrides support** (`snpm-overrides.yaml`, `package.json`)
- [x] **Peer dependency handling** with configurable strictness

## Planned

Upcoming features and improvements.

### Performance
- [ ] Smarter reuse of resolution graphs within a workspace
- [ ] Incremental lockfile updates for faster `add`/`remove`

### Developer Experience
- [ ] `snpm why <package>` to explain dependency paths
- [ ] `snpm prune` to remove extraneous packages
- [ ] Interactive upgrade mode

### Publishing
- [ ] `snpm publish` with workspace support
- [ ] `snpm pack` to create tarballs

### Advanced Features
- [ ] Side effects cache for install scripts
- [ ] Pluggable lifecycle hooks
